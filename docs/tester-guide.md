# MountBehave 测试人员使用教学

适用版本：`v0.2.4`

本文面向参与实机测试的用户，目标是让测试人员可以独立完成安装、连接、手控、星图 GOTO、校准、日志导出和问题反馈。MountBehave 当前仍是测试版，不是 OnStep 官方 App，也不适合无人值守控制。

## 1. 测试前安全检查

测试前先完成这些检查：

1. 赤道仪、三脚架、配重、镜筒和线缆已经可靠固定。
2. 镜筒转动路径内没有人、墙、三脚架腿、电脑线、供电线或其他障碍。
3. 已经知道实体断电、控制盒急停或赤道仪原控制器停止运动的方法。
4. 初次测试请使用低速或中速移动，不要直接使用最高速。
5. 夜间测试时，先在白天或室内空载熟悉界面和停止按钮。
6. 不要把当前版本用于无人值守摄影、远程自动化或无人看护的长时间 GOTO。

出现异常声音、线缆拉紧、镜筒接近三脚架、赤道仪方向明显错误时，立即使用实体急停或断电。App 内“急停/停止”是辅助安全手段，不能替代实体安全措施。

## 2. 准备工作

测试人员需要准备：

- 一台 Android 手机或平板，建议 Android 8.0 或更新版本。
- `MountBehave-v0.2.4.apk`。
- 一台可通过 WiFi TCP 控制的 OnStep / OnStepX 赤道仪。
- 赤道仪 WiFi 名称、密码、IP 和端口。默认常见值是 `192.168.0.1:9999`。
- 观测地经纬度，或允许手机使用定位权限。
- 用于反馈问题的日志导出方式，例如 QQ、微信、邮箱或电脑 USB 文件传输。

建议测试时记录：

- 手机/平板型号和 Android 版本。
- 赤道仪型号、固件类型和固件版本。
- 使用经典 OnStep 还是 OnStepX。
- 赤道仪模式：赤道仪或经纬仪。
- 测试地点大致区域，不需要公开精确住址。

## 3. 安装 APK

### 从 GitHub Release 安装

1. 在手机或平板上下载 `MountBehave-v0.2.4.apk`。
2. 打开 APK。
3. 如果系统提示“禁止安装未知来源应用”，允许当前浏览器或文件管理器安装未知来源应用。
4. 安装完成后打开 MountBehave。

### 通过电脑安装

如果使用 ADB：

```powershell
adb install -r MountBehave-v0.2.4.apk
```

`-r` 表示覆盖安装，通常可以保留 App 数据。若测试前需要完全清空设置和日志，可以在系统设置里清除 MountBehave 的应用数据。

## 4. 首次打开后的设置

打开 App 后先进入“设置”页。

### 固件和支架模式

1. 如果使用经典 OnStep，保持默认固件模式即可。
2. 如果使用 OnStepX，先选择 OnStepX。
3. OnStepX 下再选择支架模式：
   - 赤道仪：德国赤道仪、叉式赤道仪等需要赤经/赤纬语义的设备。
   - 经纬仪：Alt-Az 架台，不需要极轴精调，也不需要架台侧处理。

切换支架模式是高风险设置。不了解固件当前配置时，不要随意写入。

### 命令日志

“设置”页底部有“命令日志”区域：

- 默认勾选“记录日志”。
- 勾选后日志区域展开并开始记录。
- 未勾选时日志区域折叠，App 不记录普通日志。

测试期间建议保持勾选。出现问题后不要先清空日志，先导出。

### 安全与夜视

“安全与夜视”区域包含：

- 全局急停。
- 夜视模式。
- 取消 GOTO。
- 刷新 GOTO 状态。
- Park / Unpark。

`Set Home` 和 `Return Home` 已经移除。OnStep 的 Home 是机械轴参考，不是普通天球目标，也不是 Park 的替代品。

## 5. 连接赤道仪

### 手机连接赤道仪 WiFi

1. 打开 Android 系统 WiFi 设置。
2. 连接赤道仪 WiFi 热点。
3. 如果系统提示“此 WiFi 无互联网，是否保持连接”，选择保持连接。
4. 有些手机会自动切回有互联网的 WiFi 或移动数据。测试时请确认当前仍连接在赤道仪 WiFi。

电脑模拟器通常不能直接加入赤道仪自己的 WiFi 热点。真实连接测试应使用手机或 Android 平板。

### App 内连接

1. 打开 MountBehave。
2. 进入“连接/同步”页。
3. 在“支架连接”里确认 IP 和端口。默认是：

```text
192.168.0.1
9999
```

4. 点击晴空角标连接按钮。
5. 连接成功后，页面会显示已连接状态，并显示断开按钮。

如果连接失败：

- 确认手机仍连接赤道仪 WiFi。
- 确认 IP 和端口正确。
- 确认赤道仪控制器已经开机并完成 WiFi 启动。
- 关闭手机的自动切换网络或“智能网络切换”。
- 尝试断开后重新连接。

## 6. 同步地点和时间

连接后建议先同步观测地和时间。

### 使用 GPS

1. 进入“连接/同步”页。
2. 点击“使用GPS”。
3. 系统请求定位权限时选择允许。
4. 等待页面显示 GPS 或网络位置。
5. 点击“同步时间/地点”。

### 手动输入坐标

1. 在纬度和经度输入框填写观测地坐标。
2. 纬度范围是 `-90` 到 `90`。
3. 经度范围是 `-180` 到 `180`。
4. 点击“同步时间/地点”。

夏令时和时区由 Android 系统规则处理。测试人员只需要确认手机系统时间正确。

## 7. 手动移动测试

进入“手控”页。

### 基础移动

1. 先选择较低移动速度，例如 `1x`、`8x` 或居中速度。
2. 按住方向键，赤道仪应开始移动。
3. 松开方向键，赤道仪应停止。
4. 依次测试东、西、南、北四个方向。
5. 再测试斜向按钮，确认组合方向能正常停止。

### 停止按钮

测试这些停止入口：

- 方向键松手停止。
- 手控页中间停止按钮。
- 悬浮急停按钮。
- 设置页全局急停。

测试停止时请听电机声音，确认不是只更新了 UI 而没有真正停止。

## 8. 星图使用

进入“星图”页。

### 基本操作

- 单指拖拽：移动视野。
- 双指缩放：改变视场。
- “图层”：开关星座线、太阳系、星团、星云、星系、小行星、彗星。
- “时间”：设置星图时间，或点击“现在”恢复当前时间。
- “查找目标”：输入恒星、深空天体、小天体名称，或输入 RA/Dec 坐标。

恒星和银河背景固定显示，不能在图层里关闭。银河背景来自 NASA SVS Deep Star Maps 2020，只作为目视方位参考。

### 星图时间注意事项

星图时间可以手动设置，方便检查未来或过去的天区。但只要执行会移动赤道仪的操作，例如 GOTO、同步、校准，App 会自动把星图时间恢复为现在。

这样做是为了避免把历史或未来时刻的月亮、行星、小天体坐标发送给真实赤道仪。

## 9. GOTO 测试流程

初次测试 GOTO 时，不要直接选择低空目标、过中天目标、北天极附近目标或离障碍很近的目标。建议先选择一颗中高空亮星。

### 普通 GOTO

1. 确认赤道仪已经连接。
2. 确认赤道仪未 Park。
3. 在星图中点选目标，或使用“查找目标”搜索。
4. 观察顶部状态：
   - 目标：显示目标名称。
   - GOTO：显示空闲、已发、移动中、接近目标等状态。
   - 方位：显示高度角、方位角和中天提示。
5. 点击“Goto 目标”。
6. 等待赤道仪移动。
7. 到位后，状态应变为类似“GOTO：接近 XXX”。
8. 只有到位或停止后，再选择下一个目标 GOTO。

### 连续 GOTO

测试连续 GOTO 时：

1. 先 GOTO 到目标 A。
2. 等状态变成“接近 A”。
3. 选择目标 B。
4. 再点击 GOTO。
5. 观察是否能正常发送并移动到目标 B。

如果状态显示“移动中”但赤道仪已经不动，先等待 10-20 秒。v0.2.4 会额外读取 RA/Dec 并判断是否到位或静止。如果仍无法恢复，点击“取消”，然后导出日志反馈。

如果状态提示“GOTO：需手控恢复”，说明 App 连续检测到赤道仪已停住、离目标仍很远，并且 OnStep 报告限位或硬件类错误。此时不要继续反复点 GOTO，先用手控低速把赤道仪移出限位/危险位置，确认方向键可以正常移动后再重新选择目标。

### 北天极附近 GOTO

v0.2.4 仍包含北天极附近目标卡住问题的修复。测试这类目标时请更谨慎：

- 确认镜筒和线缆不会接近极限。
- 先用低速或中速熟悉设备行为。
- 目标可以选择 Polaris 附近，但不要在设备机械限位不明确时反复大角度 GOTO。
- 如果赤道仪机械停在天极附近，App 应能在确认位置接近或稳定后释放 GOTO 状态。

反馈这类问题时，请记录目标名称、发生时间、App 顶部 GOTO 状态和赤道仪是否还在发声或移动。

## 10. 星图同步

星图同步适合在 GOTO 到位后做局部修正：先让目标进入目镜，再手动居中并同步当前位置。

1. 进入“星图”页。
2. 选择一颗实际能看到的目标，建议先用中高空亮星。
3. 点击“Goto 目标”，等待状态显示接近目标。
4. 如果目镜里仍有偏差，用方向键把目标精确居中。
5. 点击“同步”。
6. 同步成功后，在同一片天区再试一次 GOTO。

保存过两星/三星模型后，App 会把同步后的残差加入本地多点修正模型，后续 GOTO 会自动参考这些校正点。若同步后 GOTO 仍明显偏差，请导出日志，并说明同步星、目标星、偏差方向和大致偏差量。

## 11. 两星 / 三星校准

两星和三星校准用于建立 OnStep 指向模型。建议先熟悉星图 GOTO 和星图同步，再测试多星校准。

### 选星建议

- 选择能清楚看到、容易识别的亮星。
- 不要选择太低的星，建议高度角高于 25°。
- 不要选择非常接近北天极的星作为初次校准星。
- 两颗或三颗星之间应有足够角距离，不要全部挤在同一小片天区。
- 避免刚好贴近中天、树木、建筑或线缆风险区域。

### 操作步骤

1. 进入“手控”页。
2. 在“支架校准”里选择“两星校准”或“三星校准”。
3. 点击“开始校准”。
4. 点击“设置目标”，从输入框、推荐亮星或星图中选择当前校准星。
5. 用方向键把星移动到目镜中心。
6. 点击“居中后接受”。
7. 如果第一颗星需要赤道仪先转到另一侧，App 会弹出确认；确认镜筒、相机和线缆空间安全后点击“开始转动”。
8. 等待 GOTO 状态显示接近目标，再用方向键精确居中，并再次点击“居中后接受”。
9. 对下一颗校准星重复步骤 4-8。
10. 接受最后一颗星后，App 会自动请求保存模型，不需要再手动点击保存。

### 结束校准

“结束”按钮用于结束当前 App 侧校准流程。OnStep 中已经接受的点不会由手机端撤回。若校准流程异常，请先导出日志，再根据赤道仪状态决定是否重新开始或重启控制器。

## 12. 极轴精调

极轴精调只用于赤道仪模式，并且需要先完成三星校准并保存模型。经纬仪模式不显示这个流程。

操作逻辑：

1. 完成三星校准。
2. 选择“极轴精调”。
3. 选择合适的精调星。
4. 点击“1. Goto 精调星并居中”。
5. 赤道仪 GOTO 到位后，用方向键把这颗星放回目镜中心。
6. 点击“2. Refine PA 后调极轴”。
7. 赤道仪会移动到极轴误差修正位置。
8. 之后不要再用方向键，只调整赤道仪方位角/高度角螺丝，把同一颗星移回中心。

如果不了解 Refine PA 的物理流程，请先不要测试这个功能。错误使用可能让校准模型或机械姿态变得更乱。

## 13. 小行星 / 彗星

设置页有“小行星 / 彗星”分组。

当前内置 17 颗基线小天体：

- 小行星：Ceres、Pallas、Juno、Vesta、Hebe、Iris、Flora、Metis、Hygiea、Parthenope、Eunomia、Eros。
- 彗星：1P/Halley、2P/Encke、9P/Tempel 1、67P/Churyumov-Gerasimenko、109P/Swift-Tuttle。

使用方式：

1. 在“图层”里打开“小行星”或“彗星”。
2. 设置页可下载亮小行星。
3. 彗星建议逐颗按名称或编号添加。
4. 在线下载需要互联网；如果手机连接在赤道仪 WiFi 且无互联网，可能需要切换网络后再下载。

小天体亮度和位置适合目视寻星参考，不适合高精度测量或掩星预报。

## 14. 日志导出和反馈

出现问题后，请先导出日志。

### 保存到下载目录

1. 进入“设置”页。
2. 确认“记录日志”仍然勾选。
3. 点击“导出日志…”。
4. 选择“保存到下载目录”。
5. 文件会保存到：

```text
Download/MountBehave/
```

文件名类似：

```text
mountbehave-20260501.txt
```

实际文件名会带导出时间，例如 `mountbehave-20260501-150518.txt`。

这种方式通常更容易被华为文件管理、QQ 和电脑 USB 文件传输识别。

### 分享日志

如果 QQ 或其他 App 能直接接收文件，也可以在导出菜单中选择“分享”。如果分享失败，请改用“保存到下载目录”，再从文件管理器手动发送。

### 问题反馈模板

反馈问题时请尽量包含：

```text
App 版本：v0.2.4
手机/平板型号：
Android 版本：
赤道仪型号：
固件类型：OnStep / OnStepX
支架模式：赤道仪 / 经纬仪
连接方式：WiFi，IP/端口：
发生时间：
操作步骤：
目标名称或坐标：
App 当时显示的状态：
赤道仪实际动作：
是否点击过取消/急停/断开：
附件：导出的日志 txt
```

好的反馈应能让开发者按步骤复现。只说“不能用”通常无法定位问题。

## 15. 建议测试清单

每次新版本测试可以按这个顺序走一遍：

1. 安装 APK，打开 App，确认版本。
2. 旋转手机/平板屏幕，确认 App 不崩溃。
3. 连接赤道仪 WiFi。
4. App 内连接默认端口。
5. 使用 GPS 或手动坐标，同步时间/地点。
6. 测试四方向手动移动和松手停止。
7. 测试急停。
8. 在星图选择一颗亮星，未连接/已连接状态下分别确认按钮语义。
9. GOTO 一颗中高空亮星。
10. 到位后继续 GOTO 第二颗星。
11. 测试 GOTO 进行中“Goto 目标”是否变为“取消”，并确认它能停止本地 GOTO 状态。
12. 测试北天极附近或小角度 GOTO，观察是否会卡住。
13. 做一次星图同步。
14. 做一次两星校准。
15. 如果条件允许，再做三星校准和极轴精调。
16. 保存日志到下载目录，并确认能通过 QQ 或电脑找到。

不需要每位测试人员都测试所有高风险项目。初次测试优先完成连接、手控、停止、普通 GOTO 和日志导出。

## 16. 常见问题

### 连接失败

先检查手机是否仍连接赤道仪 WiFi。很多 Android 手机会因为赤道仪 WiFi 无互联网而自动切走。

### 点 GOTO 后很快回到空闲

请确认已经连接赤道仪，目标不是地平线以下，赤道仪未 Park，命令日志中是否有 `:MS#` 拒绝回复。导出日志反馈。

### GOTO 显示移动中但赤道仪已经不动

等待几次轮询。v0.2.4 会用 RA/Dec 到位复核和静止检测兜底。如果仍无法恢复，点击“取消”，导出日志。

### GOTO 提示需手控恢复

这通常表示连续 GOTO 触发了 OnStep 限位或硬件类错误，赤道仪停止但离目标仍很远。先不要继续发送 GOTO，用手控低速移动确认赤道仪已离开限位/危险位置；手控移动会清除 App 侧的保护状态。低空或地平线下目标不会触发这层保护。

### 平板旋转后连接断开

v0.2.4 已修复这个问题。若仍复现，请记录旋转前后的页面、是否正在 GOTO、是否正在移动，并导出日志。

### 日志在手机里能看，电脑或 QQ 找不到

优先使用“导出日志…”里的“保存到下载目录”。文件应在 `Download/MountBehave`，扩展名是 `.txt`。不要只使用系统文件选择器保存到某些 App 私有目录。

### GPS 不可用

可以手动输入经纬度。测试 GOTO 和校准时，观测地错误会影响方位、中天提示和同步到赤道仪的站点信息。

### 星图时间和真实天空不一致

点击星图页“时间”，再点“现在”。如果手机系统时间错误，请先修正 Android 系统时间。

## 17. 测试边界

当前版本重点验证目视手控流程：

- WiFi 连接。
- 手动移动和停止。
- 星图选星与 GOTO。
- 星图同步、两星/三星校准。
- 日志导出。

当前不建议重点测试：

- 无人值守自动化。
- 长时间无人看护跟踪。
