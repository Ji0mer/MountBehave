# MountBehave

<p align="center">
  <img src="docs/images/mountbehave_icon.png" alt="MountBehave app icon" width="150">
</p>

MountBehave 是一个面向 OnStep 赤道仪的 Android 手控器项目，目标是在手机或平板上完成赤道仪连接、手动移动、离线星图、目标 GOTO、观测地/时间同步、跟踪控制和目视校准辅助。

<p align="center">
  <img src="docs/images/clearsky_wordmark.png" alt="Clearsky ST17 test badge" width="220">
</p>

> 测试状态：目前大部分功能仍处于开发验证阶段，尚未经过完整实机测试。到目前为止，只使用晴空谐波赤道仪 ST17 完成过 WiFi 连接和手动移动/停止测试；GOTO、两星/三星校准、跟踪、Park/Unpark、限位/过中天提醒和三星极轴精调仍需要继续夜间实机验证。

## 更新说明

- 校准流程已改为更适合大偏差目视初始状态的方式：先选择目标星，再由用户手动移动赤道仪把目标星居中，最后同步或接受当前位置。
- 快速同步不再先执行 GOTO，而是“设为同步目标 → 手动居中 → 居中后同步”。
- 两星/三星校准不再先执行 GOTO，而是“开始模型 → 选择校准星 → 手动居中 → 同步接受校准星”。
- 两星/三星接受校准星时使用 OnStep Align 模式下的同步命令 `:CM#`，避免纯手动居中时误用依赖 GOTO 目标的接受命令。
- 三星极轴精调流程调整为：完成并保存三星模型后，先 GOTO 精调星，再执行 Refine PA，随后只调赤道仪方位角/高度角把星移回中心。

## 本地环境

本目录已经配置了项目内便携工具链，位于 `.toolchain/`：

- JDK 17
- Android SDK 命令行工具
- Android SDK Platform 36
- Android SDK Build Tools 36.0.0
- Android Platform Tools
- Gradle Wrapper，基于 Gradle 9.3.1 生成

在本目录打开 PowerShell 后可运行：

```powershell
.\scripts\doctor.ps1
.\scripts\build-debug.ps1
```

调试 APK 输出位置：

```text
app\build\outputs\apk\debug\app-debug.apk
```

安装到已连接的 Android 设备：

```powershell
.\scripts\env.ps1
adb devices
adb install -r app\build\outputs\apk\debug\app-debug.apk
```

在当前电脑上用本地 Android 模拟器预览：

```powershell
.\scripts\preview-app.ps1
```

预览脚本使用 `720x1280` 的紧凑模拟器显示，并以 65% 比例显示，方便普通桌面完整看到窗口。

## 当前功能

App 目前已经实现以下核心功能：

- WiFi TCP 连接 OnStep 命令端口，默认 `192.168.0.1:9999`
- `:GVP#` 连接握手
- 下拉选择手控移动速度：1x、8x、20x、48x、半最高速和最高速
- 八方向手动移动：上、下、左、右、东北、西北、东南、西南
- 松开方向键、点击停止、断开连接、App 进入后台时安全停止
- 离线星图，包含恒星、深空天体和星座连线
- 星图拖动、缩放、目标点选和目标搜索
- 未连接时显示目标，连接后可发送 GOTO
- 默认波士顿观测地，也支持手动输入和 GPS 获取
- 当前时间、时区和夏令时由系统规则处理
- 观测地与时间同步到 OnStep
- 悬浮菜单切换设置、手控和星图页面
- 横屏和平板宽屏适配
- 悬浮全局急停按钮，常驻在菜单按钮下方
- GOTO 状态查询与取消 GOTO
- Park/Unpark 停放控制
- 夜视红色界面和低亮度显示
- 星图目标高度、方位、低空/地平线下限位和过中天提醒
- 手控页集成校准区，可用下拉菜单选择快速同步、两星校准、三星校准或三星极轴精调
- 快速同步、两星/三星 OnStep 指向模型建立流程
- 保存 OnStep 指向模型
- 设置页启动跟踪：三星校准后自动请求模型补偿双轴，否则使用单轴
- Refine PA 极轴精调提示流程
- 自定义 MountBehave 启动器图标

## 使用教程

### 1. 安装和启动

1. 在本目录运行 `.\scripts\build-debug.ps1` 构建 APK。
2. 用 USB 连接手机并打开开发者模式，或启动本地模拟器。
3. 运行 `adb install -r app\build\outputs\apk\debug\app-debug.apk` 安装。
4. 打开 MountBehave。App 默认进入“设置”页。

电脑模拟器通常不能直接扫描或加入赤道仪自己的 WiFi 热点；真实观测时建议把 APK 安装到手机或平板上使用。

### 2. 连接赤道仪

1. 先让手机连接到赤道仪 WiFi。
2. 在“赤道仪连接”里确认 IP 和端口。默认是 `192.168.0.1:9999`。
3. 点击“连接”。连接成功后会显示 OnStep 握手信息或“无握手返回”提示。
4. 如果连接异常，先确认端口是否为 `9999`，再尝试断开后重新连接。

### 3. 设置观测地与时间

1. 在“观测地与时间”中使用 GPS、波士顿默认值，或手动输入经纬度。
2. 点击“同步观测地与时间到赤道仪”，把当前地点、时间和时区发送给 OnStep。
3. 夏令时和时区由 Android 系统规则处理，用户通常只需要确认经纬度正确。

### 4. 手控移动

1. 从左上角菜单切换到“手控”页。
2. 选择移动速度：`1x` 适合精细居中，`8x/20x/48x` 适合找目标，半最高速和最高速适合大范围移动。
3. 按住方向键移动，松开即停止。中间红色“停止”和悬浮“急停”都会发送全停止命令。

### 5. 星图和 GOTO

1. 切换到“星图”页，可拖动和双指缩放天球视图。
2. 点击星图目标，或点击“Goto 目标/查找目标”输入天体名称或 RA/Dec 坐标。
3. 未连接时只在星图显示目标；已连接时可以选择显示并 GOTO。
4. GOTO 后可用“刷新 GOTO 状态”查看赤道仪是否仍在移动，也可以随时“取消 GOTO”。
5. 星图会显示目标高度、方位，并在低空、地平线下或接近中天时给出提醒。

### 6. 跟踪和校准

1. 在“设置 → 跟踪”里选择恒星速、月球速或太阳速。
2. 未完成三星校准时，启动跟踪默认使用单轴。
3. 完成三星校准或三星极轴精调后，再启动跟踪会请求 OnStep 使用模型补偿双轴。
4. “手控”页校准区可选择快速同步、两星校准、三星校准或三星极轴精调。
5. 快速同步适合临时目视：选择亮星，手动移动赤道仪把它居中，然后点击“居中后同步”。
6. 两星/三星校准按“开始校准 → 选择校准星 → 手动居中 → 同步接受校准星”的流程逐颗进行。
7. 完成两星/三星校准星接受后，点击“保存模型”请求 OnStep 保存指向模型。
8. 三星极轴精调需要先完成并保存三星模型，再 GOTO 精调星，执行 Refine PA 后只用赤道仪方位角/高度角调节螺丝把星移回中心，不要用方向键。

### 7. 安全与夜视

1. “安全与夜视”里可以执行全局急停、取消 GOTO、Park/Unpark 和夜视模式。
2. Park/Unpark 使用 OnStep 标准命令 `:hP#` 和 `:hR#`。
3. 夜视模式会把主界面改为暗红色并降低屏幕亮度，适合现场目视观测。

## 无赤道仪测试

如果手边没有真实 OnStep 赤道仪，可以启动本地模拟服务：

```powershell
.\scripts\mock-onstep.ps1
```

然后在 Android 模拟器中连接：

```text
10.0.2.2:9999
```

如果使用真机连接，则把 IP 改成本电脑在局域网中的地址，端口保持 `9999`。
